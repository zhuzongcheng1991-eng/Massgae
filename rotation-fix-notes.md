# 技师轮班逻辑修复说明

## 问题描述
之前的轮班逻辑存在问题：每天的第一个订单没有正确地从"前一天最后一个订单的下一个技师"开始，导致轮班不公平。

## 修复内容

### 1. 核心逻辑修改

**函数：`updateRotationStateForDate(targetDate)`**

这个函数在每次打开新订单模态框时调用，用于更新轮班状态。

逻辑：
1. 查找目标日期之前的所有已完成订单
2. 按日期和完成时间排序，找到最后一个完成的订单
3. 检查是否有比已记录的 `lastDate` 更新的完成订单
4. 更新 `rotationState.lastTherapistId` 为最后一个完成的技师
5. 设置 `rotationState.lastDate = targetDate`

**函数：`getStartingTherapistForNewDay()`**

计算第二天应该从哪个技师开始：
```javascript
return (lastTherapistId % 技师总数) + 1
```

示例：
- 如果最后完成的是 1 号技师 → 次日从 2 号开始
- 如果最后完成的是 5 号技师 → 次日从 1 号开始（循环）

### 2. 实时更新轮班状态

**函数：`updateRotationStateAfterCompletion(date, therapistId)`**

当订单自动完成时（`checkAndCompleteServices`），实时更新轮班状态：
- 记录当天最后一个完成的技师
- 保存到 localStorage

### 3. 测试文件

创建了多个测试文件：

1. **test-rotation.html** - 基础逻辑测试
2. **test-rotation-full.html** - 完整场景测试（包含休息日）
3. **test-rotation-simple.html** - 简化测试页面
4. **generate-march-data.html** - 3 月数据生成器

## 测试步骤

### 方法 1：使用测试页面
1. 打开 `test-rotation-simple.html`
2. 查看测试结果，验证每天的第一单推荐是否正确

### 方法 2：使用主应用
1. 打开 `generate-march-data.html`，点击"生成数据"
2. 刷新 `index.html`
3. 切换到不同日期，点击"新建订单"
4. 查看控制台日志，验证推荐的技师是否正确

### 方法 3：手动测试
1. 清空数据
2. 创建 3/30 的订单，分配给 1-5 号技师，标记为 completed
3. 切换到 3/31，点击"新建订单"，应该推荐 1 号技师（王师傅）
4. 创建 3/31 的订单，分配给 1-3 号技师
5. 切换到 4/1，点击"新建订单"，应该推荐 4 号技师（刘师傅）

| 3/31 | 5 号 陈师傅 | 1 号 王师傅 |

## 控制台日志示例

打开浏览器控制台，应该看到类似以下的日志：

```
[轮班] 更新：最后完成订单是 2026-04-02 技师 4
[轮班] 今天应从 5 号开始
[轮班更新] 新的一天，记录技师 5
[轮班更新] 更新今天最后技师为 5
```

## 注意事项

1. 只有状态为 `completed` 的订单才会计入轮班
2. 休息日（无订单）不影响轮班逻辑，次日会从最近一个有订单的日期的最后技师开始
3. 轮班状态保存在 localStorage 中，刷新页面后仍然有效
